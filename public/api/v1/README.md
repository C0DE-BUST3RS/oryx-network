# Waar is dit project voor

Dit PHP-project is ervoor om een RESTful API te hebben voor bijvoorbeeld een Android app of een Single Page Application ([wiki](https://nl.wikipedia.org/wiki/Single_Page_Application)).

_Het is een voorbeeldproject, dus je kunt er nog veel meer mee doen ;) !_


# Hoe start ik de api

## Voorkennis

### Apache+PHP

Van jou wordt verwacht dat je weet hoe je een php project kunt uitvoeren via bijvoorbeeld WAMP of XAMPP.
Plaats dus de inhoud van dit project met alle bestanden in een eigen mapje zodat je via de browser het project kunt vinden.

## Composer

Uitgaande van een windows systeem: Installeer [Composer](https://getcomposer.org/) eerst (als je dit nog niet gedaan hebt).

Ga naar de command prompt (CMD) en navigeer naar de map van dit project.

Voer hier het commando: `composer update` uit.
Deze zorgt ervoor dat je de bibliotheken krijgt die nodig zijn voor dit project aan de hand van het `composer.json` bestand.

**(Tip: Wanneer succesvol uitgevoerd, ga opzoek naar het `vendor` mapje)**


De bibliotheken die toegepast worden voor dit project zijn:

* [jacwright/restserver](https://packagist.org/packages/jacwright/restserver)

* Een specifiek Artikel uit de database: [http://localhost/[jouw mapje]/articles/[articleId]](http://localhost/[jouw mapje]/articles)


## WAMP connectieproblemen 

Wanneer jij lokaal wel de api kunt zien (dus: [http://192.168.0.107/[jouw mapje]/articles/](http://localhost/articles/)) maar in LogCat de volgende error ziet:

```
04-24 14:43:28.593 13476-13566/nl.novacollege.ictacademie.amo.android.androidexamples D/OkHttp: <-- 403 Forbidden http://192.168.0.107/android-test-api/articles (18ms)
    Date: Tue, 24 Apr 2018 12:43:31 GMT
```

Dan dien je de volgende handelingen te verrichten om Wamp toe te staan de data vanaf het LAN ook te serveren.

### Open `httpd-vhosts.conf` 

Via het wamp menu (linker muisknop) via `Apache` => `httpd-vhosts.conf`

### Wijzig

`Require local` naar `Require all granted` 

Origineel:

```
# Virtual Hosts
#
<VirtualHost *:80>
  ServerName localhost
  ServerAlias localhost
  DocumentRoot "${INSTALL_DIR}/www"
  <Directory "${INSTALL_DIR}/www/">
    Options +Indexes +Includes +FollowSymLinks +MultiViews
    AllowOverride All
    Require local
  </Directory>
</VirtualHost>
```
 
Nieuwe versie:

```
# Virtual Hosts
#
<VirtualHost *:80>
  ServerName localhost
  ServerAlias localhost
  DocumentRoot "${INSTALL_DIR}/www"
  <Directory "${INSTALL_DIR}/www/">
    Options +Indexes +Includes +FollowSymLinks +MultiViews
    AllowOverride All
    Require all granted
  </Directory>
</VirtualHost>

```

### Firewall (Windows)

Zorg dat in de windows firewall een "inkomende regel" / "Inbound rule" toegevoegd wordt waar TCP poort 80 toegelaten wordt.


Bron: [WAMP Cannot access on local network 403 Forbidden](https://stackoverflow.com/questions/23382627/wamp-cannot-access-on-local-network-403-forbidden)
